Bridge Spatial Ontology (BSO)

Version 1.0 OWL 2 Turtle (TTL) Licence: CC BY 4.0

A spatial ontology for automated bridge BIM component assembly using WBS-based spatial relationships. BSO formally encodes the hierarchical structure, spatial coupling relationships, and assembly semantics of rigid-frame bridge components, enabling SPARQL-driven automated placement within BIM authoring tools.

Cha, H. & Moon, H. (2026). A Spatial Ontology-Based Framework for Automated Bridge BIM Component Assembly: Focus on Rigid-frame Bridges. Automation in Construction (under review).

Competency Questions

The following questions can be answered by querying the BSO ontology with SPARQL:

#Competency QuestionKey Properties Used
CQ1What components are vertically attached to a given pier footing?bso:isAttachedTo
CQ2Which components are horizontally adjacent to an abutment wall?bso:isAdjacentOf
CQ3What is placed on top of the slab?bso:isPutOn
CQ4Which substructure components bear the superstructure?bso:isBearingWith
CQ5What is the full vertical assembly sequence of Pier A1?bso:isAttachedTo (chain)
CQ6Which components belong to a specific pier or abutment?bso:isPartOf
CQ7What material is used for a given component?bso:material
CQ8What is the station location of all piers?bso:staLocation

SPARQL Query Examples

CQ1/CQ5: Pier Vertical Assembly Sequence

PREFIX bso: <https://hyunwoochacha.github.io/SO-ASAS/ontology#>

SELECT ?footingName ?foundationName ?columnName ?copingName
WHERE {
    bso:A1_PierFoundation_Instance bso:isAttachedTo bso:A1_PierFooting_Instance .
    bso:A1_PierColumn_Instance     bso:isAttachedTo bso:A1_PierFoundation_Instance .
    bso:A1_PierCoping_Instance     bso:isAttachedTo bso:A1_PierColumn_Instance .

    bso:A1_PierFooting_Instance    bso:name ?footingName .
    bso:A1_PierFoundation_Instance bso:name ?foundationName .
    bso:A1_PierColumn_Instance     bso:name ?columnName .
    bso:A1_PierCoping_Instance     bso:name ?copingName .
}

CQ3: Components Placed on the Slab

PREFIX bso: <https://hyunwoochacha.github.io/SO-ASAS/ontology#>

SELECT ?componentName
WHERE {
    ?component bso:isPutOn bso:Slab_Instance .
    ?component bso:name ?componentName .
}

CQ4: Substructure Components Bearing the Superstructure

PREFIX bso: <https://hyunwoochacha.github.io/SO-ASAS/ontology#>

SELECT ?componentName ?componentType
WHERE {
    ?component bso:isBearingWith bso:Slab_Instance .
    ?component bso:name ?componentName .
    ?component rdf:type ?componentType .
    FILTER(?componentType != owl:NamedIndividual)
}

CQ6: All Members of Pier A1

PREFIX bso: <https://hyunwoochacha.github.io/SO-ASAS/ontology#>

SELECT ?member ?memberName ?memberType
WHERE {
    ?member bso:isPartOf bso:A1_Pier_Instance .
    ?member bso:name ?memberName .
    ?member rdf:type ?memberType .
    FILTER(?memberType != owl:NamedIndividual)
}

CQ8: Station Locations of All Piers

PREFIX bso: <https://hyunwoochacha.github.io/SO-ASAS/ontology#>

SELECT ?pier ?station
WHERE {
    ?pier rdf:type bso:Pier .
    ?pier bso:staLocation ?station .
}
ORDER BY ?station
